# Impeccable Project Context

## Product
Healthcare risk analytics and decision-support platform. Analysts upload patient cohort CSVs, run batch ML risk scoring (via FastAPI backend), and explore results through dashboards covering KPIs, segmentation, calibration, fairness metrics, and pricing simulation. Role-based access splits Researcher vs Customer views.

## Users
Medical insurance underwriters, healthcare data scientists, and risk compliance analysts. They expect precision, data density, and professional credibility. Not consumer-facing — this is a B2B analytics tool.

## Visual Style
Clinical and analytical. Data-dense but organized. Should feel like a professional SaaS analytics product (think Metabase or Looker). Neutral palette with purposeful color for data encoding. No decorative elements.

## Colors
Primary: Blue (#2563EB) — primary actions, selected states, links
Secondary: Slate (#475569) — labels, axis text, metadata
Accent: Amber (#D97706) — warnings, calibration flags, risk thresholds
Positive: Emerald (#059669) — low-risk indicators, success states
Negative: Rose (#E11D48) — high-risk flags, errors
Background: Slate-50 (#F8FAFC) / Slate-900 (#0F172A) in dark
Text: Slate-800 (#1E293B) / Slate-200 (#E2E8F0) in dark

## Typography
System-sans (Inter) for labels. Tabular numerics (font-variant-numeric: tabular-nums) for all data values. Compact but readable — 13–14px for table cells, 15px for body. Avoid decorative or display fonts entirely.

## Layout & Spacing
Sidebar nav (11 routes) + main content area. Dashboard uses a responsive multi-column grid for KPI cards and charts. Tables are scrollable with sticky headers. Tight 4–8px spacing within data components. 16–24px between sections.

## Components
- CSV upload with schema validation feedback
- KPI card grid (metric + delta + sparkline)
- Recharts: bar, line, scatter, calibration curve, histogram
- Segmentation explorer with filterable cohort table
- Bootstrap CI confidence interval display
- Fairness metrics table with parity thresholds
- Pricing simulator with sliders
- Role switcher (Researcher / Customer)
- Run status indicator + batch progress
- Report export button

## UX Principles
Trust is paramount — every metric needs a label with units. Charts must have axis labels and tooltips. No metric appears without context (benchmark or threshold). Data loads synchronously where possible; async states use skeleton loaders. Errors on scoring API calls are specific and recoverable.

## Anti-patterns to Avoid
- Unlabeled chart axes or missing units
- Color alone to convey risk meaning (must add text/icon)
- Hiding data behind extra clicks when it fits on screen
- Decorative gradients or glassmorphism on data-heavy views
- Auto-refreshing charts that reset user scroll position

## Notes for AI Edits
Analytics computation logic in `src/lib/` is off-limits. Chart types and data bindings are intentional — do not swap chart libraries. Safe to change: colors, spacing, typography, card layout, sidebar styling. Role-based visibility rules must be preserved exactly.
